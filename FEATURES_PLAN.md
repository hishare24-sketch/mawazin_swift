> تحديثات وثيقة واجهات الاستخدام (UI/UX) — تُنفَّذ مرحليًا فوق المعمارية الحالية (Vue 3 + Vuetify + Pinia + localStorage) بمحاكاة ذكاء اصطناعي مقنعة عبر طبقة مجرّدة قابلة للتبديل لـ Claude API.

## المرحلة 1 — المهارات كإثبات موثّق

- نموذج: `Skill { proofs[], selfLevel, confidence(derived), aiLevel(derived) }`، `SkillProof { type, refId, weight, date }`.
- أوزان الثقة: اختبار +35، توصية +25، مشروع +20، شهادة +15، ذاتي +5 (سقف 100).
- مكوّنات: أيقونات الإثبات، صف مهارة بشريط ثقة، `AddProofDialog`, `SkillDetailDialog`.
- خصوصية: ظهور المهارات/الإثباتات/نسبة الثقة/السماح بطلب إثبات.
- AI: اقتراح المستوى + طلب إثبات مهارة.

## المرحلة 2 — نسبة الثقة (Trust Score)

- محرّك 8 عوامل موزونة يقرأ من المخازن: اكتمال 20% · توصيات 25% · اختبارات 20% · إثبات مهارات 15% · تفاعل 5% · حداثة 5% · نشاط 5% · مقابلات 5%.
- مكوّنات: `TrustScoreCard` (gauge)، `TrustRadar` (رادار SVG)، `TrustBreakdownDialog`، `TrustTips` (نصائح AI قابلة للتنفيذ).
- استخدام: شارة الثقة + فلتر/ترتيب المرشحين حسب الثقة.

## المرحلة 3 — نظام المقابلات وتحديد المستوى

- وحدة `modules/interviews/`: لوحة + إعداد + جلسة AI تفاعلية + صفحة نتيجة (تُحدّث Trust Score) + سجل.
- الأنواع: AI نصية/فيديو، خارجية، خبير. المستويات: أساسي/متوسط/متقدم/خبير (مع مقابل اختياري).
- جهة التوظيف: زر "طلب مقابلة" + فلتر حسب مستوى المقابلة.

## المرحلة 4 — الربط النهائي

- المساعد: تفعيل الوظائف الأربع (تحليل الثقة، اقتراح مقابلة، طلب إثبات، تحليل المقابلات).
- الإشعارات: طلب إثبات، دعوة مقابلة، تغيّر نسبة الثقة.
- الملف: قسم المقابلات المُنجزة + تحديث المهارات والثقة.

## المرحلة 5 — سوق المقيّمين المعتمدين (Certified Interviewers Marketplace)

> منصّة داخل المنصّة: خبراء معتمدون (مقيّمون) يُجرون مقابلات تقييمية موثّقة للمرشحين مقابل مقابل مادّي، وتُغذّي نتائجها **نسبة الثقة** وملف المرشح. تبني على وحدة `interviews` الحالية (نوعا «خبير/خارجية» يصبحان مدعومَين بمقيّمين حقيقيين) وعلى نمط السوق من `requests`.

### القرارات المعتمدة (توصيات)
- **دور جديد من الدرجة الأولى:** `interviewer` يُضاف إلى `UserRole` (auth + register selector + navigation + guards). لوحته وتدفّقاته منفصلة.
- **الربط بالمقابلات الحالية:** المقابلات بنوع `expert`/`external` تصبح الواجهة الأمامية لهذا السوق؛ مقابلات الـ AI الذاتية تبقى كما هي. يُعاد استخدام `InterviewResult` + تكامل Trust بدل التكرار.
- **المحاكاة:** غرفة المقابلة (فيديو/صوت) والمدفوعات **محاكاة بصرية فقط** (اتّساقًا مع نطاق المشروع — بلا WebRTC/بوابة دفع).
- **تغذية الثقة:** تقارير المقيّمين تُغذّي عامل «المقابلات» في محرّك الثقة (نفس الـ8 عوامل) عبر `InterviewsStore.trustValue`؛ بلا تغيير الأوزان.

### النماذج والمخازن
- `InterviewersStore`: `Interviewer { id, name, specialty(type), title, bio, rating, reviewsCount, sessionsCount, priceRange, availability[], badges[], eligibility }`، تقييمات، أوقات متاحة. أنواع المقيّم: تقني/إداري-قيادي/نفسي-سلوكي/متخصّص.
- `BookingsStore` (أو توسيع `InterviewsStore`): حجز `{ interviewerId, candidateId, interviewKind, datetime, price, status: requested|scheduled|completed|cancelled, report? }`.
- `EvaluationReport { level, competencies[], strengths[], improvements[], recommendation, trustGain }` (يُعاد استخدام شكل `InterviewResult`).
- أنواع المقابلة (سوقيًا): تقييم مهارات · تحديد مستوى · قيادية · شخصية/سلوكية · شاملة — بمقابل يحدّده المقيّم.

### عقود الذكاء الاصطناعي الجديدة (تُضاف لطبقة `services/ai`)
```ts
interviewerEligibility(quals)      // نسبة الأهلية المبدئية + نقاط قوة/ضعف + توصية قبول/رفض
suggestInterviewerPricing(type, quals) // اقتراح سعر حسب السوق والمؤهلات
interviewerMatch(candidate, interviewer) // نسبة التوافق
recommendInterviewers(candidate)   // أفضل 3 مقيّمين للمرشح
suggestEvaluationQuestions(kind, candidate) // أسئلة مخصّصة للمقابلة
reviewEvaluationReport(reportDraft) // مراجعة/تلخيص تقرير المقيّم + استخراج نقاط
```

### الشاشات الجديدة (`src/modules/interviewers/`)
- **جانب المرشح:** سوق المقيّمين (`/interviewers` — بطاقات + فلترة تخصّص/سعر/تقييم/توفّر + «موصى بهم لك») · ملف المقيّم (`/interviewers/:id` — مؤهلات/تقييمات/نسبة توافق + حجز) · حجز مقابلة (نوع + وقت + تأكيد سعر) · سجل مقابلاتي (موجود، يُوسّع بتقارير المقيّمين).
- **جانب المقيّم (دور `interviewer`):** تسجيل «مقيّم معتمد» (معالج + نسبة أهلية AI) · لوحة تحكّم (حجوزات/طلبات/أرباح/تقييمات/إدارة أسعار) · غرفة المقابلة (نموذج تقييم + أسئلة AI مقترحة) · تقرير المقابلة (يُراجعه AI ويُضاف لملف المرشح ويحدّث الثقة).
- **جانب الجهة:** استعراض تقارير مقابلات المرشحين · طلب مقابلة لمرشح عبر مقيّم معيّن · (لاحقًا) مقابلة جماعية.
- **الخصوصية (إضافات):** ظهور تقارير المقابلات · ظهور التسجيلات · السماح بطلب مقابلة من الجهات/المقيّمين.
- **التوثيق:** تقرير نصّي · تسجيل (محاكاة) · تحليل AI · شهادة رقمية «اجتياز مقابلة».

### الترتيب التنفيذي (4 دفعات)
1. **الأساس:** دور `interviewer` (النوع + التسجيل + التنقّل + الحُرّاس) + `InterviewersStore` بمقيّمين وهميين + عقود AI الستّة.
2. **جانب المرشح:** سوق المقيّمين + ملف المقيّم + الحجز → يظهر في سجل المقابلات ويغذّي الثقة.
3. **جانب المقيّم:** معالج التسجيل (أهلية AI) + لوحة التحكّم + غرفة المقابلة + التقرير.
4. **جانب الجهة + الخصوصية + التقارير:** تقارير في ملف المرشح + شهادة رقمية + إعدادات الخصوصية + طلب الجهة عبر مقيّم.

> **نموذج الأعمال (عرض فقط):** عمولة 10–20% · اشتراك مقيّمين · توصيات مميّزة · شهادات — تُعرض كإحصائيات/أرباح في لوحة المقيّم بلا معالجة دفع فعلية.
