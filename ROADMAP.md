# خطة تنفيذ منظومة التوظيف الذكية متعددة الأبعاد

> خارطة طريق التنفيذ على مراحل — مبنية على الوثيقة المعمارية، ملف البنية (`DOC/PROJECT_STRUCTURE.md`)، ووثيقة واجهات الاستخدام (`DOC/FILES/`).

---

## القرارات المعتمدة

| القرار | الخيار المعتمد | السبب |
|--------|----------------|-------|
| إطار الواجهة الأمامية | **Vue 3 + Vuetify 3** | حسب `PROJECT_STRUCTURE.md` (دليل تطوير مفصّل بأنماط جاهزة) — اعتُمد بدل React المذكور في الوثيقة المعمارية |
| نطاق هذا المستودع | **الواجهة الأمامية فقط (Web)** | الـ API والـ AI في مستودعات منفصلة |

> [!NOTE]
> الوثيقة المعمارية ذكرت `React + Tailwind`، لكن ملف البنية يصف `Vue 3 + Vuetify`. اعتُمد **Vue** لأنه الدليل العملي الأكثر تفصيلًا. تُعامل إشارة React في الوثيقة المعمارية كنصّ قديم.

## 1) معمارية المستودعات

النظام موزّع على 5 طبقات، والمقترح **Multi-repo**:

| المستودع | المحتوى | التقنية |
|----------|---------|---------|
| `recruitment-api` | بوابة API، المصادقة، منطق الأعمال، توليد PDF/DOCX | Laravel + Sanctum + Echo |
| **`smart-recruitment-system`** (هذا الريبو) | لوحة التحكم + بوابة الباحثين/الشركات | Vue 3 + Vuetify |
| `recruitment-ai` | المطابقة، NLP، منشئ السيرة، تحليل الاستبيانات | Python + FastAPI + HuggingFace |
| `recruitment-mobile` | تطبيق الجوال (مراحل لاحقة) | Flutter |

**طبقة البيانات:** MySQL + Redis + S3/MinIO + Meilisearch (بحث).

## 3) المراحل التنفيذية

### 🟦 المرحلة 0 — الأساس المتين (≈ شهرين) — MVP

**الهدف:** نظام يعمل من طرف لطرف: تسجيل → ملف → نشر فرصة → تقديم → سيرة أساسية.

| الطبقة | المخرجات |
|--------|----------|
| Backend | إعداد Laravel، MySQL، Sanctum، RBAC (`view_*/create_*`)، Migrations للكيانات الأساسية (User, Profile, Opportunity, Application, Endorsement بسيطة)، رفع ملفات (S3/MinIO)، توليد PDF (DomPDF) |
| Frontend (هذا الريبو) | تهيئة Vite + Vuetify + قالب Vuexy، الـ Layouts الثلاثة، Axios + interceptors (401/توكن)، i18n (ar/en + RTL)، AuthStore، حُرّاس المسارات، وحدات: `auth`, `profile`, `opportunities`, `applications`, `team`، نسخة أساسية من `resume-builder` (قالب واحد + PDF) |
| AI | لا شيء بعد (أو خدمة وهمية للمطابقة ترجع نسبة ثابتة) |
| DevOps | GitHub Actions: lint + typecheck + build، بيئات `.env.development/production` |

**معيار الإنجاز (DoD):** باحث ينشئ ملفًا ويقدّم على فرصة بسيرة PDF؛ شركة تنشر فرصة وتشاهد المتقدمين؛ مدير يدير المستخدمين.

---

### 🟩 المرحلة 1 — الذكاء والتقييم (≈ 3–4 أشهر)

**الهدف:** محرك المطابقة، الاختبارات، التوصيات المتقدمة، المساعد، ومنشئ السيرة المتكامل.

| الطبقة | المخرجات |
|--------|----------|
| AI (`recruitment-ai`) | محرك المطابقة (AI Match %)، NLP لتحليل التوصيات، **منشئ السيرة الكامل** (5 قوالب، عربي/إنجليزي/ثنائي، إعادة صياغة الملخص والإنجازات)، Survey AI، منع الغش (Proctoring أولي) |
| Backend | كيانات: Assessment, Resume (template/sections/shareable_token)، Endorsement (صوت/فيديو)، توليد DOCX (PHPWord)، تكامل مع خدمة الـ AI، WebSockets (Echo) |
| Frontend | وحدات: `assessments` (إجراء اختبار + مؤقت + تحليل)، `endorsements` (تسجيل صوت/فيديو + أسئلة AI)، `assistant` (محادثة كاملة + اقتراحات سياقية)، **`resume-builder` الكامل** (معالج 7 خطوات + معاينة حية + إعادة صياغة + DOCX + رابط مشاركة)، عرض تحليل التطابق التفصيلي في صفحة الفرصة |

**DoD:** ترشيحات بنسبة تطابق حقيقية؛ سيرة بخمسة قوالب وتصدير PDF/DOCX ورابط عام؛ المساعد يجيب ويقترح؛ توصيات صوت/فيديو.

---

### 🟨 المرحلة 2 — التوسع والتفاعل (≈ 6–8 أشهر)

**الهدف:** العرض المزدوج، الاستبيانات الثمانية، سوق المهمات، تحليلات السوق.

| الطبقة | المخرجات |
|--------|----------|
| Backend | Wish (عرض مزدوج + تفاوض)، Survey كامل (8 أنواع + إرسال خارجي بريد/واتساب + معرّف خارجي للمستجيب)، سوق المهمات الجزئية، تكامل LinkedIn API |
| Frontend | وحدات: `wishes` (واردة/مرسلة + تفاوض)، `surveys` (إنشاء/إجابة داخل وخارج المنصة + تحليل)، `tasks-market`، `analytics` (لوحة سوق المواهب + رسوم بيانية + تصدير PDF/Excel/CSV) |
| AI | تحليل استبيانات تلقائي، اقتراح كلمات مفتاحية ATS، مطابقة السيرة مقابل متطلبات الفرصة |

**DoD:** جهة ترسل رغبة ويتفاوض الباحث؛ استبيان يُرسل لمستفيد خارجي بلا حساب ويُحلَّل؛ لوحة تحليلات سوق فعّالة.

---

### 🟧 المرحلة 3 — النضج والاستدامة (سنة فأكثر)

| الطبقة | المخرجات |
|--------|----------|
| الأعمال | نموذج الإيرادات (اشتراكات/عمولات)، إدارة الباقات والفواتير، نظام السمعة والمكافآت |
| التقنية | فتح API لطرف ثالث، تكامل ATS عالمي، سير ذاتية تفاعلية عبر الرابط، تحليلات تنبؤية متقدمة، **تطبيق Flutter** |

## 4) المسار الحرج — أول 6 خطوات عملية

```
1. حسم إطار الواجهة (Vue ✅)
2. تهيئة هذا الريبو: Vite + Vuetify + Vuexy + i18n(RTL) + Axios + AuthStore
3. إنشاء ريبو recruitment-api (Laravel + Sanctum + RBAC)
4. تعريف عقد الـ API (OpenAPI) للكيانات الأساسية ← مرجع مشترك
5. بناء وحدة auth كاملة (نمط المرجع participant-categories)
6. بناء وحدة profile + opportunities ← أول رحلة كاملة E2E
```

## 6) مؤشرات الأداء (KPIs)

- نسبة التطابق بين الترشيحات والفرص (هدف > 70%)
- متوسط وقت إغلاق الفرصة (هدف < 10 أيام)
- صافي نقاط الترويج NPS (هدف > 50)
- عدد السير الذاتية المنشأة شهريًا
- معدل استخدام المساعد الذكي وإكمال الاستبيانات
